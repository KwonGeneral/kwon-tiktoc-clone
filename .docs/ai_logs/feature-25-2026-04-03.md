# Feature 25: 디바이스 UUID + 영상/이미지 삭제

- **날짜**: 2026-04-03
- **브랜치**: feature/phase-25-uuid-delete

## 작업 요약

### 1. UUID 서비스 구현
- `uuid` 패키지 추가
- `DeviceIdService` 생성: 앱 최초 실행 시 UUID v4 생성 → SharedPreferences 저장
- `deviceIdServiceProvider`로 DI (main.dart에서 override)

### 2. 하드코딩 userId 교체 (9곳)
- `AppStrings.commentCurrentUserId = 'current_user'` → 디바이스 UUID
- 교체 대상:
  - `profile_provider.dart` (프로필 이미지 업로드, 내 영상/이미지 필터링)
  - `publish_provider.dart` (영상 업로드)
  - `publish_image_provider.dart` (이미지 업로드)
  - `comment_provider.dart` (댓글 작성)
  - `feed_provider.dart` (업로드 영상 추가)
  - `user_repository_impl.dart` (getCurrentUser mock 데이터)
  - `video_remote_datasource.dart` (업로드 fallback userId 제거)
  - `post_image_remote_datasource.dart` (업로드 시 userId 파라미터화)

### 3. 삭제 기능 구현 (Clean Architecture 전 레이어)
- **Domain**: `VideoRepository.deleteVideo()`, `PostImageRepository.deletePostImage()` 추가
- **Data**: `DELETE /videos/:id?userId=`, `DELETE /videos/images/:id?userId=` API 연동
- **Presentation**: 프로필 그리드 길게 누르기 → 삭제 확인 다이얼로그 → API 호출 + Optimistic UI 갱신

### 4. 서버 테스트 데이터 정리
- video id 13 (userId=current_user) 삭제
- image 1개 (userId=current_user) 삭제
- id 14~18은 이미 삭제된 상태
