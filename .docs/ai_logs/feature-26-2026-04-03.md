# Feature 26: 랜덤 닉네임 + 멀티유저 지원

- **날짜**: 2026-04-03
- **브랜치**: feature/phase-26-random-nickname-multiuser

## 작업 내용

### 1. 랜덤 닉네임/username 생성기
- `lib/core/utils/random_nickname_generator.dart` 신규 생성
- 한글 닉네임: 형용사 + 동물/채소 + 4자리 숫자 ("귀여운고양이#3847")
- 영문 username: adjective + noun + 4자리 숫자 ("cute_cat_3847")

### 2. LocalStorage 확장
- `LocalStorageRepository`에 `getProfileUsername()`, `saveProfileUsername()` 추가
- `LocalStorageService`에 구현

### 3. User Entity/Model 확장
- `User` entity에 `username` 필드 추가
- `UserModel`에도 동일하게 추가 + toEntity/fromEntity 반영

### 4. 최초 실행 시 자동 생성
- `currentUserProvider`에서 nickname/username이 비어있으면 자동 생성 + 저장

### 5. "나의계정" 하드코딩 제거
- `user_repository_impl.dart`의 mock user nickname을 빈 문자열로 변경
- Provider에서 로컬 프로필 닉네임으로 override

### 6. 업로드 시 프로필 정보 전송
- 영상 업로드: username을 별도로 가져와서 전송 (기존 nickname과 분리)
- 이미지 업로드: nickname/username 파라미터 추가 (Domain → Data → DataSource 전체)

### 7. 프로필 화면
- ProfileHeader에서 @username 표시 (기존 @deviceId 대체)

### 8. 게시 시 설명 필수 입력
- 동영상/이미지 게시 페이지에서 설명이 비어있으면 게시 버튼 비활성화
- `disabledBackgroundColor`/`disabledForegroundColor` 명시

### 9. 앱바 중복 게시 버튼 제거
- 앱바의 게시 버튼 제거, 타이틀("게시")로 교체
- 하단 게시 버튼만 유지

## 변경 파일 수
- 신규: 1개
- 수정: 15개+ (Entity, Model, Repository, DataSource, Provider, View, Constants)
