# Fix 21F: HLS 전환 + 썸네일 + 비디오 최적화

- **날짜**: 2026-04-03
- **브랜치**: feature/fix-21f-hls-thumbnail
- **작업 시간**: ~15분

## 작업 내용

### 1. Video Entity + Model에 hlsUrl 필드 추가
- `video.dart`: hlsUrl 필드 추가 (Default '')
- `video_model.dart`: hlsUrl 필드 + fromEntity/toEntity 매핑
- build_runner 재생성

### 2. VideoRemoteDataSource에서 hlsUrl, thumbnailUrl 파싱
- API 응답의 `v['hlsUrl']`, `v['thumbnailUrl']`을 VideoModel에 매핑
- thumbnailUrl은 이미 Entity에 존재했으나 파싱되지 않았음 → 추가

### 3. HLS 재생 + MP4 fallback
- `video_player_manager.dart`: hlsUrl이 있으면 HLS 우선 사용, 없으면 videoUrl(MP4) fallback
- 업로드된 새 영상은 HLS 미생성 가능 → fallback 필수

### 4. 로딩 중 썸네일 이미지 표시
- `video_card.dart`의 `_buildLoading()`: thumbnailUrl이 있으면 Image.network로 표시
- 검정 화면 대신 첫 프레임 이미지 → UX 개선

### 5. 프리로딩 범위 확대
- ±1 → ±2로 변경 (keepRange 루프 offset -2~+2)
- 더 넓은 범위의 컨트롤러 유지 → 스와이프 시 즉시 재생

### 6. 업로드 반환값 변경
- DataSource/Repository/Provider 전체: `Future<void>` → `Future<Video>`
- 서버 응답 JSON 파싱하여 VideoModel 반환
- MockDataSource도 동일하게 변경
