# Fix 21I: 남은 버그 일괄 수정

- **날짜**: 2026-04-03
- **브랜치**: `fix/phase-21i-bug-fixes`
- **모델**: Claude Opus 4.6

## 작업 내용

### 1. 좋아요/북마크 딜레이
- 코드 검토 결과 이미 optimistic update가 구현되어 있음 (state 즉시 변경 후 비동기 서버 호출)
- 추가 수정 불필요

### 2. 댓글창 애니메이션
- `video_card.dart`에 `AnimationController` + `CurvedAnimation` 추가
- 300ms easeOutCubic 커브로 영상 축소 + 댓글 슬라이드 업 애니메이션
- 닫기 시 reverse 애니메이션 (easeInCubic)
- 닫기 버튼은 progress 30% 이후부터 fade in

### 3. 팔로잉 탭 필터링
- `displayVideos` getter 필터 로직 검증 완료
- `followedUserIds` 기반 필터링이 정상 동작
- API 모의 데이터에서 각 user_N이 1개 영상만 보유하여 1명 팔로잉 시 1개 영상만 표시되는 것은 정상

### 4. 게시 영상 유저명/음악정보 UUID → 닉네임
- `publish_provider.dart`: `username` 필드를 deviceId → 프로필 닉네임으로 변경
- `feed_provider.dart` `addUploadedVideo`: username, nickname, musicName에 프로필 닉네임 사용
- `feed_provider.dart` `_applyLocalStates`: 기존 업로드 영상의 UUID 유저명을 감지하여 닉네임으로 교체
- `_looksLikeUuid` 헬퍼 메서드 추가 (UUID v4 형식 검사)

### 5. 유저명 텍스트 오버플로우
- `video_description.dart`: 유저명 Text를 Flexible로 래핑 + maxLines: 1 + TextOverflow.ellipsis

### 6. 영상 삭제 후 홈 무한로딩
- `video_player_manager.dart`: 영상 목록 크기 변경 감지 → 모든 컨트롤러 dispose 후 재초기화
- `feed_provider.dart` `deleteVideo`: 삭제 후 currentIndex가 범위를 벗어나지 않도록 clamp
- `feed_page.dart`: 영상 목록 변경 시 PageController 재생성
