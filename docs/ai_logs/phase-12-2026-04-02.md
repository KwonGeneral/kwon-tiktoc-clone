# Phase 12: API 연동 - AI 대화 로그

- **날짜**: 2026-04-02
- **Phase**: 12 - API 연동
- **브랜치**: feature/phase-12-api-integration

---

## 작업 내용

### 1. http 패키지 추가
- pubspec.yaml에 `http: ^1.3.0` 추가

### 2. Video Entity/Model 필드 확장
- `username`, `nickname`, `avatarUrl` 3개 필드를 Video Entity와 VideoModel에 추가
- fromEntity/toEntity 매핑 업데이트
- build_runner로 freezed/json_serializable 코드 재생성

### 3. VideoRemoteDataSource 구현
- `GET https://api.myfortie.com/videos` API 호출
- API 응답 구조 분석: 13개 영상, metadata에 userId/username/nickname/avatarUrl 포함
- API의 metadata.tags를 해시태그 형식으로 변환하여 description에 포함
- 캐싱 적용 (한 번 로드 후 메모리에 보관)
- 좋아요/북마크 토글은 로컬 캐시에서 처리
- 댓글은 기존 Mock 방식 유지

### 4. Provider 재구성
- `providers.dart`에서 MockVideoDataSource → VideoRemoteDataSource로 교체
- DI 체인 유지: DataSource → Repository → UseCase → UI

### 5. 피드 UI 업데이트
- VideoDescription: `@user_{userId}` → 실제 `@{username}` 표시
- SideActionBar: 기본 아이콘 → 실제 avatarUrl 네트워크 이미지 표시 (fallback 포함)

## 주요 결정사항
- API 응답의 video URL은 `hd/` 경로 사용 (기존 Mock은 `full-hd/`)
- API의 id는 int(0~12)이므로 `video_0` ~ `video_12` 형태로 변환
- 좋아요/댓글/북마크/공유 수는 API에 없으므로 기존 Mock 방식 유지 (index 기반 생성)
- musicName도 API에 없으므로 기존 패턴 유지
