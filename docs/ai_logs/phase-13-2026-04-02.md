# Phase 13 - 로컬 DB + 상태 영속화

- **날짜**: 2026-04-02
- **브랜치**: feature/phase-13-local-db-state-persistence

## 작업 요약

### 목표
좋아요/북마크/팔로우 상태를 로컬에 영속화하여 앱 재시작 후에도 유지

### 구현 내용

1. **shared_preferences 패키지 추가**
   - key-value 저장소로 충분, hive보다 가벼운 선택

2. **LocalStorageService 구현** (`data/datasource/local_storage_service.dart`)
   - SharedPreferences 래퍼 클래스
   - liked_video_ids, bookmarked_video_ids, followed_user_ids 키로 Set<String> 저장/로드

3. **DI 설정** (`core/di/providers.dart`)
   - `localStorageServiceProvider` 추가 (override 패턴)

4. **main.dart 초기화**
   - SharedPreferences.getInstance() → LocalStorageService 생성 → Provider override

5. **FeedNotifier 연동** (`presentation/feed/provider/feed_provider.dart`)
   - `_applyLocalStates()`: API 비디오에 저장된 좋아요/북마크 상태 적용
   - `toggleLike/toggleBookmark/toggleFollow`: 토글 시 로컬 저장
   - `loadMore`: 새 비디오에도 로컬 상태 적용

### 설계 결정
- Domain 레이어 변경 없음 (Clean Architecture 준수)
- 낙관적 업데이트 패턴 유지 + 로컬 저장 추가
- API 응답에 로컬 상태를 오버레이하는 방식

### 변경 파일
- `pubspec.yaml` — shared_preferences 추가
- `lib/data/datasource/local_storage_service.dart` — 신규
- `lib/core/di/providers.dart` — localStorageServiceProvider 추가
- `lib/main.dart` — 초기화 + Provider override
- `lib/presentation/feed/provider/feed_provider.dart` — 상태 영속화 연동
