# Phase 15: 공유하기 + 채널 이동

**날짜**: 2026-04-02
**모델**: Claude Opus 4.6

## 작업 요약

### 구현 항목
1. `share_plus` 패키지 추가 (v10.1.4)
2. 공유 버튼 → `Share.share()`로 시스템 공유 시트 호출 (영상 URL + 설명 텍스트)
3. `UserProfilePage` 구현 — 아바타, @username, 팔로잉/팔로워/좋아요 통계, 팔로우 토글, 유저 영상 그리드
4. `UserVideoGrid` 위젯 — 3열 그리드로 유저 영상 표시
5. GoRouter에 `/user/:userId` 라우트 추가 (ShellRoute 밖 — 하단 네비 없음)
6. `SideActionBar`의 프로필 아바타/팔로우 뱃지 탭 분리
   - 아바타 탭 → 유저 프로필 페이지로 이동
   - 뱃지 탭 → 팔로우 토글
7. `VideoOverlay`에 공유/프로필 콜백 연결

### 주요 변경 파일
- `pubspec.yaml` — share_plus 추가
- `lib/app/route/route_paths.dart` — userProfile 경로 + 헬퍼 메서드
- `lib/app/route/app_router.dart` — /user/:userId 라우트
- `lib/core/constants/app_strings.dart` — 공유/유저프로필 문자열
- `lib/presentation/feed/widget/side_action_bar.dart` — onShareTap, onProfileTap 콜백 + 아바타/뱃지 분리
- `lib/presentation/feed/widget/video_overlay.dart` — Share.share(), context.push 연결
- `lib/presentation/user_profile/view/user_profile_page.dart` — 신규
- `lib/presentation/user_profile/widget/user_video_grid.dart` — 신규

### 설계 결정
- UserProfilePage는 feedNotifierProvider에서 해당 userId의 영상을 필터링하여 사용 (별도 API 없이)
- 유저 프로필은 ShellRoute 밖에 배치하여 하단 네비게이션 없이 전체 화면으로 표시
- 아바타 탭과 팔로우 뱃지 탭을 GestureDetector로 분리하여 독립적 동작
